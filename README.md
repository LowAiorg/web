# LowAI · web

Public site for **LowAI** + a serverless, privacy-first **supporters** system.

## Pages
- `lowai.html` — the manifesto / landing.
- `Supports.html` — the public supporters list, stats, and self-service sign / remove.

## How "I support it" works (no server we maintain)

Two verifiable identity channels, both self-managed:

### 🐙 GitHub
- The button opens a **pre-filled issue** (`?labels=support&template=support.yml`).
- The issue author **is** the verification — it's their real GitHub identity.
- A GitHub Action (`.github/workflows/supporters.yml`) rebuilds `supporters.json` + `stats.json`
  from every **open** issue labelled `support`.
- **Self-removal:** the supporter **closes their own issue**. Nobody can remove anyone else.

### 🔑 Nostr
- "Sign with Nostr" builds a `kind:1` note tagged `t:lowaisupport`, signed via a
  **NIP-07** browser extension (Alby, nos2x). **The private key never touches the page.**
- `Supports.html` reads those notes **live** from public relays (raw WebSocket, no library).
- **Self-removal:** a `kind:5` deletion event (NIP-09), signed with the same key.
- Relays enforce signature validity, so events served to the page are authentic.

## Data files
- `supporters.json` — `{ updated, supporters: [{ platform, handle, name, url, avatar, date, issue }] }`
  (GitHub channel; regenerated by the Action — do not edit by hand).
- `stats.json` — `{ updated, total, byPlatform: { github, nostr } }`
  (`nostr` is `null` here; the page counts Nostr live).

## Privacy
No backend, no database, no cookies, no trackers. The page only fetches two static JSON
files and queries public Nostr relays. Supporters reveal only the handle/key they choose.

## Roadmap
More channels (email magic link, Fediverse, Web3 wallet) can be added as extra collectors
feeding the same `{platform, handle, date}` shape — the per-platform stats already account for them.
